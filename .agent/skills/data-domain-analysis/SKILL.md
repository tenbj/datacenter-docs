# 数据域分析技能

## 概述

此技能用于帮助分析数据仓库中的数据域，包括：
- 业务过程对比分析
- 粒度定义合理性评估
- 维度指标矩阵检查

## 使用场景

当用户提出以下类型的请求时，应使用此技能：
1. "帮我分析XX域的业务过程"
2. "对比XX域和YY域的差异"
3. "检查维度指标矩阵的合理性"

## 执行步骤

### 步骤 1：获取数据域定义

首先读取权威资料中的数据域定义表：
```
权威资料/1.1 数据域定义表.md
```

### 步骤 2：收集待分析的数据域文档

根据用户指定的数据域，读取以下文件：
- `output_LLM/数据域详情/XX_XX域_XX_vX.X.md`

### 步骤 3：执行分析

根据 `resources/analysis_template.md` 模板进行分析，包含：

| 分析维度 | 检查内容 |
|----------|----------|
| 业务过程完整性 | 是否覆盖所有核心业务流程 |
| 粒度一致性 | 粒度定义是否清晰、可度量 |
| 维度合理性 | 维度是否支持业务分析需求 |
| 指标覆盖度 | 指标是否满足KPI要求 |

### 步骤 4：生成报告

使用 `resources/report_template.md` 生成分析报告，输出至：
```
output_LLM/计划书/XX域分析报告_vX.X.md
```

## 注意事项

> [!IMPORTANT]
> 所有输出文件必须遵循版本管理规则，参见 `.agent/rules/version-control-rules.md`

> [!TIP]
> 分析多个数据域时，建议使用对比表格形式呈现差异

## 示例调用

用户输入：
```
帮我分析库存域和履约域的业务过程是否有重叠
```

预期行为：
1. 读取两个数据域的详情文档
2. 提取业务过程清单
3. 进行交叉对比
4. 输出对比分析报告
