## 1. 核心数据域详解 (MECE原则业务过程)

基于《数据域业务过程梳理方法论》，遵循“原子性、可衡量、有时点、可追溯”原则，对核心 P0/P1 数据域的业务过程进行标准化拆解。

### 1.1 交易域 (Trade - trd)
*   **业务过程拆解 (原子粒度)**:
    1.  **购物车管理**: `加购` (cart_add) -> `移除` (cart_remove)
    2.  **正向交易**: `下单` (order_place) -> `支付` (payment) -> `确认` (confirm)
    3.  **交易履约**: `发货` (shipped) -> `签收` (delivered) -> `完成` (completed)
    4.  **逆向交易**: `取消` (cancel) -> `退款` (refund) -> `退货` (return)
    5.  **结算**: `结算` (settlement)
*   **关键事实表 (DWD明细层)**:
    *   `dwd_trd_cart_add_dd` (交易域_加购明细_日增量)
    *   `dwd_trd_order_detail_dd` (交易域_订单明细_日增量) - *核心表*
    *   `dwd_trd_pay_detail_dd` (交易域_支付明细_日增量)
    *   `dwd_trd_return_detail_dd` (交易域_退换货明细_日增量)
*   **关键事实表 (DWS汇总层)**:
    *   `dws_trd_sku_sales_1d` (交易域_SKU销售_1日汇总)
    *   `dws_trd_shop_sales_1d` (交易域_店铺销售_1日汇总)
*   **一致性维度**:
    *   `dim_trd_shop` (店铺), `dim_trd_site` (站点), `dim_prod_sku` (商品), `dim_pub_date` (公共日期).

### 1.2 运营域 (Operation - op)
*   **业务过程拆解 (原子粒度)**:
    1.  **流量获取**: `曝光` (impression) -> `点击` (click) -> `浏览` (view)
    2.  **内容管理**: `Listing发布` (publish) -> `Listing优化` (optimize)
    3.  **用户互动**: `评价` (review) -> `反馈` (feedback) -> `QA提问` (qa)
    4.  **账号管理**: `绩效考核` (performance_check)
*   **关键事实表 (DWD明细层)**:
    *   `dwd_op_traffic_session_dd` (运营域_流量会话_日增量)
    *   `dwd_op_review_cntnt_dd` (运营域_评论内容_日增量)
    *   `dwd_op_acct_health_dd` (运营域_账号绩效_日增量)
*   **关键事实表 (DWS汇总层)**:
    *   `dws_op_asin_traffic_1d` (运营域_ASIN流量转化_1日汇总)
*   **一致性维度**:
    *   `dim_prod_asin` (链接/ASIN), `dim_op_traffic_src` (流量来源), `dim_op_acct_status` (账号状态).

### 1.3 广告域 (Advertising - ads)
*   **业务过程拆解 (原子粒度)**:
    1.  **投放配置**: `预算设置` (budget_set), `竞价设置` (bid_set)
    2.  **广告触达**: `广告曝光` (ad_impression) -> `广告点击` (ad_click)
    3.  **广告转化**: `广告加购` (ad_cart) -> `广告订单` (ad_order)
    4.  **费用扣缴**: `广告扣费` (ad_cost)
*   **关键事实表 (DWD明细层)**:
    *   `dwd_ads_camp_perf_dd` (广告域_活动表现_日增量)
    *   `dwd_ads_srch_term_dd` (广告域_搜索词表现_日增量)
    *   `dwd_ads_bid_change_dd` (广告域_竞价变更_日增量)
*   **关键事实表 (DWS汇总层)**:
    *   `dws_ads_sku_roi_1d` (广告域_SKU投入产出_1日汇总)
*   **一致性维度**:
    *   `dim_ads_camp` (广告活动), `dim_ads_group` (广告组), `dim_ads_kw` (关键词), `dim_ads_match_type` (匹配类型).

### 1.4 库存域 (Inventory - inv)
*   **业务过程拆解 (原子粒度)**:
    1.  **入库作业**: `采购入库` (inbound), `退货入库` (return_inbound)
    2.  **库内作业**: `盘点` (cycle_count), `移库` (transfer), `状态变更` (status_change)
    3.  **出库作业**: `销售出库` (outbound), `移除出库` (removal)
    4.  **持有**: `库存持有` (holding) -> 产生库龄和仓储费
*   **关键事实表 (DWD明细层)**:
    *   `dwd_inv_receipt_dtl_dd` (库存域_入库明细_日增量)
    *   `dwd_inv_trans_dtl_dd` (库存域_库存事务_日增量)
*   **关键事实表 (DWS汇总层)**:
    *   `dws_inv_sku_snap_df` (库存域_SKU库存快照_日全量) - *注意：快照表使用df后缀*
    *   `dws_inv_strg_fee_1d` (库存域_仓储费_1日汇总)
*   **一致性维度**:
    *   `dim_inv_wh` (仓库), `dim_inv_type` (库存类型).

### 1.5 履约域 (Fulfillment - fl)
*   **业务过程拆解 (原子粒度)**:
    1.  **发货准备**: `打单` (labeling) -> `拣货` (picking) -> `打包` (packing)
    2.  **干线运输**: `离港` (departure) -> `到港` (arrival) -> `清关` (customs)
    3.  **末端配送**: `提货` (pickup) -> `派送` (delivery) -> `签收` (sign)
*   **关键事实表 (DWD明细层)**:
    *   `dwd_fl_inbound_ship_dd` (履约域_头程发货单_日增量)
    *   `dwd_fl_ship_track_dd` (履约域_物流轨迹_日增量)
*   **关键事实表 (DWS汇总层)**:
    *   `dws_fl_dlvry_time_1d` (履约域_配送时效_1日汇总)
*   **一致性维度**:
    *   `dim_fl_carrier` (物流承运商), `dim_fl_route` (航线), `dim_fl_ship_status` (物流状态).

## 2. 本次需求相关的数据域

根据《需求总览_260104》及《应用场景》文档，本次需求主要涉及以下 9 个数据域（按相关度排序）：

1.  **交易域 (trd)**: 涉及“销售毛利”、“订单结构”、“转化分析”、“业绩目标”等核心需求。
2.  **运营域 (op)**: 涉及“流量分析”、“市场趋势”、“账号健康”、“客户评价”等。
3.  **广告域 (ads)**: 涉及“广告效益”、“广告跑时”、“关键词竞价”、“广告流量”。
4.  **库存域 (inv)**: 涉及“FBA仓储”、“库存计划”、“动销分析”。
5.  **促销域 (prm)**: 涉及“活动促销效益”、“活动复盘”。
6.  **市场域 (mkt)**: 涉及“竞品分析”、“市场热度词”、“站外推广”。
7.  **财务域 (fin)**: 涉及“利润核算”、“成本费用”、“回款”。
8.  **客服域 (cs)**: 涉及“客诉指标”、“退货损失”。
9.  **履约域 (fl)**: 涉及“仓储费用预估”（偏履约成本）、“发货销量预测”。

## 4. 数据域能力覆盖 (域 -> 需求)

### 4.1 交易域 (trd) 能满足的需求
*   **业绩监控**: 销售额、销量、GMV、订单量。
*   **结构分析**: 分站点、分店铺、分品类、分变体的销售占比。
*   **转化分析**: 结合流量数据，计算购买转化率 (CR)。
*   **退款分析**: 退款金额、退款率（作为分母）。

### 4.2 运营域 (op) 能满足的需求
*   **流量分析**: PV, UV, Session, PageViews。
*   **流量来源**: 自然流量 vs 广告流量占比。
*   **Listing质量**: 评分、评论数、BSR排名监控。
*   **账号风控**: ODR, IPI, 绩效分监控。

### 4.3 广告域 (ads) 能满足的需求
*   **广告效果**: ACOS, TACOS, ROAS, CPC, CTR。
*   **投产分析**: 广告花费 vs 广告销售额。
*   **关键词分析**: 搜索词报告，精准/宽泛匹配效果。

### 4.4 库存域 (inv) 能满足的需求
*   **库存健康**: 库龄分布，滞销库存预警。
*   **补货决策**: 结合销量的当前库存水位，可售天数 (DOS)。
*   **资产分析**: 库存货值，FBA在库数量。

### 4.5 市场域 (mkt) 能满足的需求
*   **竞对分析**: 竞品销量预测，竞品价格波动。
*   **趋势分析**: 类目大盘走势，关键词搜索热度。

### 4.6 财务域 (fin) 能满足的需求
*   **利润报表**: 单品毛利，店铺净利，ROI (投资回报率)。
*   **成本管控**: 头程运费占比，仓储费占比，广告费占比。

### 4.7 促销域 (prm) 能满足的需求
*   **活动复盘**: Prime Day, 黑五网一的效果评估。
*   **优惠券分析**: Coupon领取率，核销率。

### 4.8 客服域 (cs) 能满足的需求
*   **VOC分析**: 差评内容挖掘，退货原因分布。
*   **服务指标**: 客服响应时间，24小时回复率。
