# 数据域建设细化分析报告

**版本**: v1.0
**日期**: 2026-01-12
**状态**: 正式稿

本文档针对“数据域建设范围”进行深度拆解，回答关于业务过程、事实表、维度及需求映射的核心问题。

---

## 1. 核心数据域详解 (MECE原则业务过程)

基于 MECE (相互独立，完全穷尽) 原则，对核心 P0/P1 数据域的业务过程进行拆解。

### 1.1 交易域 (Trade - trd)
*   **业务过程拆解 (按订单生命周期)**:
    1.  **正向交易**: 
        *   `下单` (Order Placed) -> `支付` (Payment) -> `发货` (Shipped) -> `完成` (Completed)
    2.  **逆向交易**: 
        *   `取消` (Cancelled) -> `退款` (Refunded) -> `退货` (Returned) -> `索赔` (Claimed)
*   **关键事实表**:
    *   `dwd_trd_order_detail_dd` (订单明细累积快照表): 涵盖下单、支付、发货等全状态时间点。
    *   `dwd_trd_return_detail_dd` (退换货明细事务表): 专注逆向流程。
    *   `dws_trd_sku_sales_1d` (SKU销售日汇总): 销量、销售额、折扣额。
*   **一致性维度**:
    *   `dim_shop` (店铺), `dim_site` (站点), `dim_sku` (商品), `dim_date` (日期), `dim_geo` (地理位置).

### 1.2 运营域 (Operation - op)
*   **业务过程拆解 (按运营对象)**:
    1.  **流量运营**: `曝光` (Impression) -> `点击` (Click) -> `转化` (Conversion) (自然流量部分)
    2.  **内容运营**: `Listing上架`, `Listing优化`, `QA问答`
    3.  **用户运营**: `Review评价`, `Feedback反馈`
    4.  **账号运营**: `绩效监控` (Account Health), `违规记录`
*   **关键事实表**:
    *   `dwd_op_traffic_session_dd` (流量会话明细)
    *   `dwd_op_review_detail_dd` (评论明细)
    *   `dws_op_asin_traffic_1d` (ASIN流量转化日汇总)
*   **一致性维度**:
    *   `dim_asin` (链接), `dim_traffic_source` (流量来源), `dim_account_status` (账号状态).

### 1.3 广告域 (Advertising - ads)
*   **业务过程拆解 (按投放漏斗)**:
    1.  **投放执行**: `预算设置`, `竞价调整`
    2.  **投放效果**: `广告曝光` -> `广告点击` -> `广告归因订单`
    3.  **费用结算**: `广告扣费`
*   **关键事实表**:
    *   `dwd_ads_campaign_performance_dd` (广告活动表现)
    *   `dwd_ads_search_term_dd` (搜索词表现)
    *   `dws_ads_sku_roi_1d` (SKU广告ROI日汇总)
*   **一致性维度**:
    *   `dim_campaign` (广告活动), `dim_ad_group` (广告组), `dim_keyword` (关键词), `dim_match_type` (匹配类型).

### 1.4 库存域 (Inventory - inv)
*   **业务过程拆解 (按物资流转)**:
    1.  **入库流程**: `采购入库`, `调拨入库`, `退货入库`
    2.  **库内管理**: `盘点`, `库龄更新`, `仓租计算`
    3.  **出库流程**: `销售出库`, `移除出库`, `调拨出库`
*   **关键事实表**:
    *   `dwd_inv_receipt_detail_dd` (入库明细)
    *   `dws_inv_sku_snapshot_df` (库存日快照 - 包含库龄)
    *   `dws_inv_storage_fee_1d` (仓储费日汇总)
*   **一致性维度**:
    *   `dim_warehouse` (仓库), `dim_inventory_type` (库存类型: 可售/不可售/预留).

### 1.5 履约域 (Fulfillment - fl)
*   **业务过程拆解 (按物流链路)**:
    1.  **头程物流**: `订舱` -> `装柜` -> `离港` -> `到港` -> `提货`
    2.  **尾程配送**: `拣货打包` -> `出库交接` -> `配送在途` -> `妥投/拒收`
*   **关键事实表**:
    *   `dwd_fl_inbound_shipment_dd` (头程发货单)
    *   `dws_fl_delivery_time_1d` (配送时效统计)
*   **一致性维度**:
    *   `dim_carrier` (物流承运商), `dim_route` (航线/线路), `dim_shipment_status` (物流状态).

## 2. 本次需求相关的数据域

根据《需求总览_260104》及《应用场景》文档，本次需求主要涉及以下 9 个数据域（按相关度排序）：

1.  **交易域 (trd)**: 涉及“销售毛利”、“订单结构”、“转化分析”、“业绩目标”等核心需求。
2.  **运营域 (op)**: 涉及“流量分析”、“市场趋势”、“账号健康”、“客户评价”等。
3.  **广告域 (ads)**: 涉及“广告效益”、“广告跑时”、“关键词竞价”、“广告流量”。
4.  **库存域 (inv)**: 涉及“FBA仓储”、“库存计划”、“动销分析”。
5.  **促销域 (prm)**: 涉及“活动促销效益”、“活动复盘”。
6.  **市场域 (mkt)**: 涉及“竞品分析”、“市场热度词”、“站外推广”。
7.  **财务域 (fin)**: 涉及“利润核算”、“成本费用”、“回款”。
8.  **客服域 (cs)**: 涉及“客诉指标”、“退货损失”。
9.  **履约域 (fl)**: 涉及“仓储费用预估”（偏履约成本）、“发货销量预测”。

## 3. 需求与数据域映射 (需求 -> 域)

| 需求大类 | 具体需求模块 | 归属数据域 | 理由 |
| :--- | :--- | :--- | :--- |
| **促销** | 活动促销效益, 活动促销费用, 站外推广 | **促销域 (prm)**, 市场域 (mkt) | 核心是活动ROI计算；站外推广属于市场获客。 |
| **竞品** | 竞品运营动态, 竞品价格, 竞品广告 | **市场域 (mkt)** | 所有外部竞争对手数据统一归口市场域。 |
| **库存** | FBA仓储, 库存计划 | **库存域 (inv)** | 涉及库存水位、库龄及补货逻辑。 |
| **价格** | 前台售价, 活动促销价, 调价 | **运营域 (op)** / 促销域 | 价格是运营手段，促销价属促销域。 |
| **广告** | 广告效益, 关键词竞价, 费用结构 | **广告域 (ads)** | 纯粹的PPC广告投放数据。 |
| **订单** | 变体订单结构, 订单来源 | **交易域 (trd)** | 订单构成的深度分析。 |
| **销售毛利** | 销售净额, 成本费用, 利润 | **财务域 (fin)** / 交易域 | 交易域提供收入数据，财务域提供成本与最终利润核算。 |
| **客服** | 客诉指标, 退货损失, 评分 | **客服域 (cs)** | 涉及VOC (Voice of Customer) 及售后。 |
| **关键词** | 自然展示位, 市场热度词 | **运营域 (op)** / 市场域 | 自然排名属运营SEO，市场热度属市场洞察。 |
| **转化** | 广告转化, 单品转化 | **交易域 (trd)** + 广告域 | 转化 = 订单 / 流量，需跨域计算。 |
| **流量** | 广告流量, 单品流量 | **运营域 (op)** + 广告域 | 流量是运营域的核心指标。 |

## 4. 数据域能力覆盖 (域 -> 需求)

### 4.1 交易域 (trd) 能满足的需求
*   **业绩监控**: 销售额、销量、GMV、订单量。
*   **结构分析**: 分站点、分店铺、分品类、分变体的销售占比。
*   **转化分析**: 结合流量数据，计算购买转化率 (CR)。
*   **退款分析**: 退款金额、退款率（作为分母）。

### 4.2 运营域 (op) 能满足的需求
*   **流量分析**: PV, UV, Session, PageViews。
*   **流量来源**: 自然流量 vs 广告流量占比。
*   **Listing质量**: 评分、评论数、BSR排名监控。
*   **账号风控**: ODR, IPI, 绩效分监控。

### 4.3 广告域 (ads) 能满足的需求
*   **广告效果**: ACOS, TACOS, ROAS, CPC, CTR。
*   **投产分析**: 广告花费 vs 广告销售额。
*   **关键词分析**: 搜索词报告，精准/宽泛匹配效果。

### 4.4 库存域 (inv) 能满足的需求
*   **库存健康**: 库龄分布，滞销库存预警。
*   **补货决策**: 结合销量的当前库存水位，可售天数 (DOS)。
*   **资产分析**: 库存货值，FBA在库数量。

### 4.5 市场域 (mkt) 能满足的需求
*   **竞对分析**: 竞品销量预测，竞品价格波动。
*   **趋势分析**: 类目大盘走势，关键词搜索热度。

### 4.6 财务域 (fin) 能满足的需求
*   **利润报表**: 单品毛利，店铺净利，ROI (投资回报率)。
*   **成本管控**: 头程运费占比，仓储费占比，广告费占比。

### 4.7 促销域 (prm) 能满足的需求
*   **活动复盘**: Prime Day, 黑五网一的效果评估。
*   **优惠券分析**: Coupon领取率，核销率。

### 4.8 客服域 (cs) 能满足的需求
*   **VOC分析**: 差评内容挖掘，退货原因分布。
*   **服务指标**: 客服响应时间，24小时回复率。
