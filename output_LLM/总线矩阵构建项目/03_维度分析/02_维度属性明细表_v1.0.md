## 1. 命名规范	

### 1.1 维度表命名	

根据数据开发规范，维度表命名采用以下格式：	
```	
dim_{域缩写}_{业务对象}	
```	

**示例**：	
- `dim_base_date` - 基础日期维度	
- `dim_trd_order_status` - 交易域订单状态维度	
- `dim_op_product` - 运营域产品维度	

### 1.2 字段命名规范	

| 字段类型 | 命名规则 | 示例 |	
|----------|----------|------|	
| 主键 | {业务对象}_key | product_key, store_key |	
| 业务编码 | {业务对象}_code 或 {业务对象}_id | product_code, store_id |	
| 名称 | {业务对象}_name | product_name, store_name |	
| 层级 | {层级名}_code, {层级名}_name | category_l1_name |	
| 标志位 | is_{标志描述} | is_active, is_deleted |	
| 日期时间 | {动作}_{时间类型} | create_time, update_date |
