## 数据域分类说明

### 按分析部门分类

#### 增长分析部门
负责业务增长相关的数据域，包括：
- **市场域** (mkt): 市场动态监测与竞争策略制定
- **交易域** (trd): 交易全流程
- **运营域** (op): 平台店铺运营
- **促销域** (prm): 促销活动
- **广告域** (ads): 广告投放管理
- **客服域** (cs): 客户服务与体验管理
- **客户域** (cst): 客户全生命周期

#### 流通分析部门
负责供应链和生产相关的数据域，包括：
- **研发域** (rd): 产品全生命周期研发
- **供采域** (sup): 供应链协同与采购执行
- **生产域** (mfg): 生产制造全过程
- **品控域** (qc): 全链路质量
- **库存域** (inv): 库存运营与优化
- **履约域** (fl): 订单交付与逆向物流

#### 价值分析部门
负责财务和政策相关的数据域，包括：
- **财务域** (fin): 财务资产与管理
- **政策域** (pol): 各类平台政策

#### 数据开发部门
负责算法和模型相关的数据域，包括：
- **算法域** (ma): 算法资产管理与智能决策支持

## 数据域应用说明

### 1. 数据域识别规则
根据表名前缀或业务特征,将表归类到对应数据域：
- 表名包含 `trd` → 交易域
- 表名包含 `mkt` → 市场域
- 表名包含 `cs` → 客服域
- 表名包含 `cst` → 客户域
- 表名包含 `fin` → 财务域
- 表名包含 `inv` → 库存域
- 表名包含 `fl` 或 `ful` → 履约域
- 表名包含 `prm` → 促销域
- 表名包含 `ads` → 广告域
- 表名包含 `op` → 运营域
- 表名包含 `sup` → 供采域
- 表名包含 `rd` → 研发域
- 表名包含 `mfg` → 生产域
- 表名包含 `qc` → 品控域
- 表名包含 `pol` → 政策域
- 表名包含 `ma` → 算法域

### 2. 数据域建设要求
每个数据域都应包含：
- **完整的业务过程**: 覆盖该域的核心业务流程
- **事实表**: DWD/DWS/DM层的事实表
- **维度表**: 支撑分析的一致性维度表
- **指标体系**: 该域的核心度量指标

### 3. 命名规范
表名应遵循以下命名规范：
```
层级_域缩写_业务对象_粒度_刷新周期
```

**示例**:
- `dwd_trd_order_main_dd`: DWD层-交易域-订单主表-日刷新
- `dws_mkt_campaign_daily`: DWS层-市场域-营销活动-天粒度
- `dim_cst_customer`: DIM层-客户域-客户维度表

### 4. 跨域关联说明
某些业务场景可能涉及多个数据域的协同：
- **交易域** + **客户域**: 客户购买行为分析
- **交易域** + **促销域**: 促销效果评估
- **广告域** + **交易域**: 广告转化分析
- **履约域** + **库存域**: 库存与物流协同
- **客服域** + **交易域**: 售后服务分析
