# 广告域业务流程与数据域一致性分析报告	

> **分析目标**: 对比权威资料中的业务流程图与数据域梳理文档，识别明显错误或不一致之处	
> **分析日期**: 2026-01-21	
> **分析版本**: v1.0	

---

## 1. 分析材料概述	

### 1.1 输入材料	

| 编号 | 材料名称           | 路径                                                                   | 核心内容                                                                |
| ---- | ------------------ | ---------------------------------------------------------------------- | ----------------------------------------------------------------------- |
| 1    | 广告创建业务流程图 | 权威资料/广告创建业务流程图_v1.0.md                                    | 描述用户创建广告的完整流程，包含投放策略、广告模式、付费类型等分支      |
| 2    | 后台操作流程图     | 权威资料/后台操作流程图_v1.0.md                                        | 描述后台计划创建流程 + 用户操作流程，包含广告类型、付费方式、定向设置等 |
| 3    | 广告域完整梳理     | output_LLM/数据域详情/数据域完整梳理_v2.0/06_P1_广告域完整梳理_v2.0.md | 广告域的业务过程、维度、事实表、总线矩阵等数据建模信息                  |

### 1.2 分析方法	

参考"产物驱动型流程架构师"方法论，按以下维度对比：	
1. **业务流程覆盖度**：数据域文档的业务流程图是否覆盖了权威流程图的关键节点	
2. **业务过程完整性**：权威流程图中的可度量事件是否都被定义为业务过程	
3. **维度覆盖度**：流程图中体现的分析维度是否都在数据域文档中定义	
4. **术语一致性**：两边使用的术语是否一致

## 2. 详细对比分析	

### 2.1 业务流程图对比	

#### 2.1.1 权威资料流程关键节点	

**广告创建业务流程图** 核心节点：	
1. 用户发起创建广告请求	
2. 创建计广告（广告名、广告类型、商品选择）	
3. 投放策略选择：自动投放 / 手动投放	
4. 手动投放下的投放位置选择：自动筛选 / 手动筛选	
5. 广告模式选择：商品模式 / 视频模式	
6. 付费类型选择：VCPM / CPC-按效果付费 / CPC-竞价量	
7. 广告类型分支：垂类 / 通投	
8. 设定投放规则、投放验据	

**后台操作流程图** 核心节点：	
1. 后台创建计划 → 选择计划属性 → 设置方案 → 选择执行方案	
2. 方案设置 → 设置子方案 → 添加子任务 → 设置投放时间	
3. 创意设置：图文创意 / 视频创意	
4. 落地页设置 → 验证链接 → 提交审核	
5. 用户选择广告类型：品牌广告 / 效果广告 / 商品广告	
6. 付费方式：CPM / CPC / OCPC	
7. 定向设置：兴趣定向 / 行为定向 / 地域定向	
8. 预算设置 → 设置日预算 → 设置出价 → 确认创建	

#### 2.1.2 数据域文档的业务流程图	

数据域文档中的业务流程图包含：	
1. 广告策略：确定广告目标 → 选择SB/SD/SP → 预算分配	
2. 广告创建：创建广告活动 → 创建广告组 → 投放类型（关键词/商品/受众）→ 设置竞价	
3. 广告投放：广告上线 → 参与竞价 → 广告曝光 → 用户行为（点击/忽略）	
4. 转化归因：进入详情页 → 是否购买 → 记录转化 → 归因计算	
5. 效果优化：数据汇总 → ACOS分析 → 效果评估 → 增加预算/优化竞价/暂停调整

#### 问题5：业务流程图风格差异	

**权威流程图特点**：	
- 从用户操作视角出发，详细描述各类选择分支	
- 涉及大量配置选项：广告名、商品链接、投放渠道、创意素材等	
- 分为"后台操作流程"和"用户操作流程"两个视角	

**数据域文档流程图特点**：	
- 从数据建模视角出发，关注可度量的事件节点	
- 更侧重于广告投放后的效果追踪（展示、点击、转化、ACOS）	
- 未详细描述广告创建前的各种配置选项	

> [!IMPORTANT]
> 这种差异本身不是错误，但数据域文档应该覆盖权威流程图中的**所有可度量节点**。

## 3. 分析结论	

### 3.1 结论汇总	

| 分析维度       | 结论                                                           | 严重程度 |
| -------------- | -------------------------------------------------------------- | -------- |
| 业务流程覆盖度 | 数据域流程图侧重投放和效果，缺少创建阶段详细步骤               | 🟡 中等   |
| 业务过程完整性 | 缺少"创意设置"、"落地页设置"、"提交审核"、"定向设置"等业务过程 | 🔴 较高   |
| 维度覆盖度     | 缺少付费类型、投放策略、创意类型、定向类型等重要维度           | 🔴 较高   |
| 术语一致性     | 付费模式术语存在差异（CPM/CPC/OCPC vs VCPM/CPC竞价）           | 🟡 中等   |
